2. 
브랜치를 만들어서 add -> commit -> push -> pr -> merge ( ---o 대중화 )

    성용 index.html을 수정 -> add -> commit -> push -> pr -> 승인 -> main branch merge
    승빈 hello.html을 수정 -> add -> commit -> push -> 
        conflict가 나는 바람에 pr에서 수정하는 법 -> 승인 -> main branch merge

협업을 위하여 pr에 제대로 된 설명을 할 필요가 있다.
처음에는 쉽지 않을 것 코드만 봐서 이게 어떤 기능인지 알 수가 없다. -> 많은 연습이 필요하다.
